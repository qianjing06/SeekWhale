
1. 项目概述
我们要开发一个名为“校园社交与数字收藏”的多人在线原生移动应用。核心玩法结合了LBS（基于位置）的社交互动与数字藏品收集。用户可以在校园地图上寻找宝箱、收集不同稀有度的藏品，并通过“日志”模块发布和参与线下活动，通过“好友”模块进行即时通讯。

2. 技术栈要求
前端：React Native (或 Flutter，根据Claude擅长的领域定，需支持跨平台)。
后端：Node.js (Express/NestJS) 或 Python (Django/FastAPI)。
数据库：MongoDB (用于用户数据、日志、藏品元数据) + Redis (用于实时位置缓存、会话管理)。
实时通信：WebSocket (Socket.IO 或类似库) 用于聊天和活动状态同步。
地图服务：集成高德地图或腾讯地图SDK（针对南京大学鼓楼校区进行地理围栏优化）。

3. 核心架构指令（重点）
用户系统：必须包含注册/登录、昵称/头像修改、ID系统。
实时交互层：所有涉及“地图宝箱”、“活动招募”和“好友聊天”的模块必须是实时的。例如，当多人宝箱被开启时，所有在范围内的用户界面必须同步刷新状态。
数据同步：用户的“展柜”数据必须实时与云端同步，防止离线丢失。

4. 模块拆解预告（请Claude做好分模块开发的准备）
请根据以下模块逐步构建系统，我会在后续指令中提供每个模块的详细UI和交互逻辑：
地图模块：包含普通宝箱（单人）和多人宝箱（协作），涉及LBS定位和解锁逻辑。
日志模块：包含发布招募、申请加入、状态流转（招募中/等待开始/结束）。
好友模块：包含好友列表、添加好友（ID搜索）、聊天界面、以及“Ta的展柜”查看。
展柜模块：用户的个人收藏展示，包含不同稀有度（典藏、神秘、限定等）的物品展示。
个人中心模块：包含个人信息编辑和收藏统计。
发布模块：包含用户发布搭子活动招募的功能。

5.身份认证系统（高优先级）
注册/登录机制：[强制] 仅支持南京大学官方邮箱（如 @smail.nju.edu.cn 或 @nju.edu.cn）进行注册和登录。
验证流程：
用户输入南大邮箱地址。
后端生成6位数字验证码，发送至该邮箱（SMTP服务）。
验证码有效期5分钟，存入Redis。
用户输入验证码，后端校验通过后创建账户或登录。
用户画像：注册成功后，自动提取邮箱前缀作为初始ID，后续可修改昵称和头像。

6. 特别强调
由于这是一个多人在线App，后端逻辑必须优先设计。请先帮我搭建好用户模型（User）、活动模型（Event）、藏品模型（Item）和好友关系模型（Friendship）的数据库Schema。
